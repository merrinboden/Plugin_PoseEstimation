BLENDER PLUGIN REFACTORING - COMPLETE ✅

The Blender plugin has been successfully refactored to work with the standalone pose estimation service.

WHAT CHANGED
============

BEFORE (Local Plugin v1.0):
- All pose detection happened inside Blender
- MediaPipe inference blocked UI during processing
- Webcam capture and gesture recognition tied to Blender

AFTER (Service Client v2.0):
- Plugin connects to separate service via WebSocket
- Blender remains responsive (no blocking)
- Service runs independently (can be on different machine)
- Plugin only receives gestures and executes actions

NEW FILES
=========
blender_client/websocket_client.py  - WebSocket connection library
blender_client/__init__.py          - Package initialization

MODIFIED FILES
==============
__init__.py:
- Updated bl_info version to 2.0.0 (Service Client)
- Simplified properties (removed webcam/smoothing/confidence settings)
- Added service_host and service_port properties
- Removed pose_estimator and gesture_handler imports

ui_panel.py:
- Updated PoseEstimationPanel to show service connection UI
- Removed camera/smoothing/debug configuration
- Added host/port input fields
- Added connection status indicator
- Updated operators to use WebSocket client
- Added gesture action handlers (pan, brush adjust, tool cycle)
- Added process_pose_updates() timer callback

USER WORKFLOW
=============
1. User starts service in terminal:
   $ python -m pose_service.server

2. User opens Blender plugin (View3D > Pose Estimation panel)

3. User enters service host/port (default: 127.0.0.1:8000)

4. User clicks "Connect to Service"
   - Plugin connects via WebSocket
   - Service starts broadcasting pose updates

5. User makes hand gestures in front of webcam
   - Service detects gestures
   - Service broadcasts gesture events
   - Blender plugin receives and executes actions

6. User clicks "Disconnect" when done

TECHNICAL DETAILS
=================
Connection:
- WebSocket: ws://127.0.0.1:8000/ws/pose (configurable)
- Non-blocking: async client runs in background thread
- Message polling: timer callback checks for updates every 16ms (~60fps)

Actions Supported:
- PAN_LEFT: Scroll camera left
- PAN_RIGHT: Scroll camera right
- PAN_UP: Scroll camera up
- PAN_DOWN: Scroll camera down
- ADJUST_BRUSH: Modify brush size (sculpt mode)
- ADJUST_STRENGTH: Modify brush strength (sculpt mode)
- SELECT_TOOL: Cycle through tools

Code Quality:
✅ Syntax validated
✅ No blocking operations in Blender
✅ Error handling with graceful fallback
✅ Clean separation of concerns (UI, networking, actions)

TESTING
=======
To test the refactored plugin:

1. Terminal 1 - Start service:
   $ python -m pose_service.server
   Output: Uvicorn running on http://127.0.0.1:8000

2. Terminal 2 - Verify service:
   $ curl http://127.0.0.1:8000/health
   Output: {"status":"ok","clients":0}

3. Blender - Load plugin:
   - Edit > Preferences > Add-ons
   - Search "Pose Estimation"
   - Enable "Pose Estimation Gesture Control (Service Client)"

4. Blender - Connect to service:
   - View3D > Sidebar > Pose Estimation tab
   - Verify Host: 127.0.0.1
   - Verify Port: 8000
   - Click "Connect to Service"

5. Move hands in front of webcam
   - Service detects poses
   - Plugin receives gestures
   - Blender viewport responds
